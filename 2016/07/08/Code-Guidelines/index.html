Code Guidelines

Posted on 2016-07-08   |   In overview

层级

代码格式

  • 缩进不允许使用Tab，只可以使用两个空格
  • 在一个规则声明中应用了多个选择器时，每个选择器独占一行
  • 在规则声明的左大括号 { 前加上一个空格。规则声明的右大括号 } 独占一行
  • 规则声明之间用空行分隔开
  • 在属性的冒号 : 后面加上一个空格。
  • 属性声明的排序
    • 首先列出除去 @include 和嵌套选择器之外的所有属性声明。
    • 紧随后面的是 @include
    • 如果有必要_用到嵌套选择器，把它们放到最后，在规则声明和嵌套选择器之间要加上空白

代码规范

  • 0替换0px。
  • 在定义无边框样式时，使用 0 代替 none。
  • 嵌套选择器
    • 请不要让嵌套选择器的深度超过 3 层！
    • 当遇到以上情况的时候，你也许是这样写 CSS 的：
      • 与 HTML 强耦合的（也是脆弱的）或者
      • 过于具体（强大）或者
      • 没有重用

命名规范

  • 类名建议使用破折号代替驼峰法。如果你使用 BEM，也可以使用下划线（参见下面的）
  • class用横线分割的小写组合：.link-style
  • ID选择器以#开头，用大写开始的驼峰形式：#App, #AppContainer。尽量不要使用 ID 选择器

特殊要求

  • 所有新增的代码，禁止使用jQuery-ui和bootstrap
  • 建议安装scss-lint和slim_lint以开启自动检查，以免不必要的格式错误
